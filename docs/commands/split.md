# csv-ops split

指定した 1 列を区切り文字列で分割し、結果を **末尾に複数列追加** する。
元の列は変更しない（flag / extract と同じく加算系。列を消したい場合は将来の `select` で行う）。

## 書式

```
csv-ops split -i <IN> -o <OUT> -c <列> --by <SEP> --out-cols <NAMES>
```

設定ファイルは持たず、CLI 引数だけで完結する。対象列は `-c` で 1 列だけ指定する。

## 分割のルール

- **区切り** — `--by` に渡したリテラル文字列で分割する（正規表現ではない）。複数文字も指定できる（例: ` - `）。
- **出力列数** — `--out-cols` に並べた列名の数で決まる。区切りの数に関わらず、常にその列数になる（出力 CSV を矩形に保つ）。
- **過剰分割**（区切りが多く、分割数が `--out-cols` の列数を超える）— 末尾の列が残りを結合して吸収する。データは失わない。
- **過少分割**（区切りが見つからない / 少ない）— 不足する列を空文字で埋める。行ごとにエラーにはしない。
- **配置** — 既存の列はそのまま、分割結果を末尾に追加する。元列も残る。

## オプション

| オプション                | 既定値  | 説明                                                                       |
| ------------------------- | ------- | -------------------------------------------------------------------------- |
| `-i`, `--input <IN>`      | -       | 入力ファイル                                                               |
| `-o`, `--output <OUT>`    | -       | 出力ファイル                                                               |
| `-c`, `--col <COLUMN>`    | -       | 分割対象の列（1 列、列名または列番号）                                     |
| `--by <SEP>`              | -       | 分割に使うリテラル区切り文字列                                             |
| `--out-cols <NAMES>`          | -       | 末尾に追加する出力列名（カンマ区切り。要素数が分割後の列数を兼ねる）       |
| `--input-encoding <ENC>`  | `utf-8` | 入力エンコーディング（utf-8 / shift_jis / euc-jp / auto）。出力は入力と同一 |
| `--delimiter <ALIAS>`     | `comma` | 区切り文字（comma / tab / pipe / semicolon）                               |
| `--no-headers`            | off     | ヘッダ行なし CSV として扱う                                                |
| `--dry-run`               | off     | 出力ファイルへ書き込まず統計のみ表示                                       |
| `--json`                  | off     | 統計を JSON 形式で出力                                                      |
| `--quiet`                 | off     | 統計出力を抑制                                                             |

## 使用例

### 氏名を姓・名に分割

入力 `in.csv`:

```
氏名,年齢
田中 太郎,30
佐藤 花子,25
```

実行:

```
csv-ops split -i in.csv -o out.csv -c 氏名 --by " " --out-cols 姓,名
```

出力 `out.csv`（元の `氏名` を残し、末尾に `姓` / `名` を追加）:

```
氏名,年齢,姓,名
田中 太郎,30,田中,太郎
佐藤 花子,25,佐藤,花子
```

### 列数が合わないとき

区切りが多い場合は末尾列が残りを吸収し、少ない場合は空文字で埋める。

```
# "田中 太郎 ジュニア" を 姓,名 に分割 → 姓="田中" 名="太郎 ジュニア"
# "佐藤"               を 姓,名 に分割 → 姓="佐藤" 名=""
```

### 中間ファイルで重ねる

分割した列をさらに別コマンドで整える運用を想定している。

```
csv-ops split -i in.csv     -o step1.csv -c 住所 --by " " --out-cols 都道府県,以降
csv-ops mask  -i step1.csv  -o out.csv   -c 都道府県
```

## 備考

- 分割は単一列のみ。複数列をまとめて分割する用途は対象外（必要なら split を複数回重ねる）。
- 元の列は削除しない。列の削除は本コマンドの責務外で、列の取捨選択は `select`（予定）で行う。
- `--by` はリテラル文字列で、正規表現は使わない（式評価エンジン非搭載の方針）。
- `--out-cols` の列名が既存カラム（分割元を含む）や他の出力列と重複する場合はエラー停止する。
- `--no-headers` 指定時はヘッダ行がないため `--out-cols` の列名は出力されず、列数だけが反映される。列指定は列番号（0 始まり）で行う。
- 統計は処理行数・ヒット行数（実際に分割が発生した = 区切りが見つかった行数）・ヒット総数を出力する。
